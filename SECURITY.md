# Security Policy

## Scope

Security reports for this repository should be limited to vulnerabilities in
this client code, tests, documentation, or Codex skill packaging.

This repository is not an official DAH channel. Reports about DAH production
systems, DAH accounts, DAH authorization, DAH data exposure, or DAH service
availability should be sent to DAH through its official support channels, not to
this repository.

## Do Not Publish Sensitive Data

Do not include any of the following in public GitHub content:

- bearer tokens, cookies, session headers, or account credentials;
- personal data, apartment data, resident data, phone numbers, or email
  addresses;
- financial records, debts, payments, invoices, bank transactions, or exported
  reports;
- association IDs, messenger group IDs, chat content, or raw API responses from
  real DAH accounts;
- screenshots that show private DAH cabinet information.

If a token or private response was exposed, revoke or rotate the credential and
remove the data from public history before opening a report.

## Supported Versions

No support or maintenance commitment is provided. Security fixes may be accepted
at the maintainer's discretion.

## Reporting

For issues in this repository's code, provide a minimal sanitized reproduction
that does not require real DAH data or credentials.

For DAH service vulnerabilities or live account issues, contact DAH through its
official support channels.
